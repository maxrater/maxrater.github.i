When working with Ruby Code, the idea is to make our code, "ebb and flow" to our will. To do this in Ruby we use enumerator's. In compuer science an enumerator is an iterator, which in our case means that the enumerator method map, sifts through an array one element at a time, and outputs each element to the code block. When I think about enumerators, I always keep in mind the each method, which is the most vanilla of enumerators, but also the most useful because you can use it for everything. In order to better understand the method map we can compare it to the each method. When the each method is run on an assortment of objects, each examines each of your elements in relation to your code block, one at a time. The method map does this same exact processing with your code. The difference between each and map is potrayed in the return value. The method Each returns its receiver, but map shoots out a new array. The new array is always the same size as the original array, but in it's place of the original elemtns, the new array containts the ammassed return values of the code block from the iterations. In conclusion, the method map is an example of a destructive method. The mapping compiles a new array, each of whose elemtns corresponds to the element in the same position in the original array but processed through the block of code. Like the method each, the method map goes through each element of the array in turn. Unlike each, map stores the value that comes back straight from the block, which is how it creates the mapping of the old values to the new values: the new values are based on the old values, treated through the block.
