Looping in Ruby vs. Looping in JavaScript

1/16/15

In programming we use looping in order to automate repetative tasks so that they might be more quickly and easily accessed. In Ruby and Javascript this is done in different ways....

In Ruby the two best ways to loop are using the 'Each' Loop and the 'While' Loop. Ruby also has a for loop (just as JavaScript does) as well as a lot of built in methods, that allow you to loop in many different ways.

Example 1 (We use the while loop)

Olympic_Medals = ["Gold", "Silver", "Bronze"] while Olympic_Medals.length == 3 p Olympic_Medals break end

Example 2 (We use the select method, one of Ruby's built in method's)

new_year_resoluion = { 1 => "Go To The Gym", 2 => "Travel", 3 => "Find Your Passion", 4 => "Screw New Years Resolutions", } new_year_resoluion.select do |key, value| p value end

Example 3

dope_music = ["keys n krates", "a-trak", "dipset", "jay-z", "diplo", "rl grime", "branchez", "gessaffelstein"] dope_music.each do |music| p "#{music}" end

In Javascript, in order to loop the syntax is slightly different...

We can use a for loop:
Example 1
for (var counter = 0; counter < 11; counter++) { console.log(counter);} This prints out numbers 0 through 10.
Example 2
var DJ = ["deadmau5", "armin van burren", "tiesto", "martin garrix"] for (var x = 0; x < DJ.length; x++){ console.log(DJ[x])} This prints out the names of the four DJ's
We can also use while loops...
A while loop runs as long as the condition is satisfied, in this occasion we set our lotteryWinner variable to true, so hence; while true, print out the string. Loops are also dangerous because they can become infinite loops, so we set lotteryWinner to equal false after printing out the string so that the loop will not go on forever and our program won't crash.
Example 3
var lotteryWinner = true while(lotteryWinner) { console.log("You have won 100 billion dollers!") lotteryWinner = false } This prints out the string: You have won 100 billion dollers
