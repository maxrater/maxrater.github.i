A class in Ruby is derived of methods that allow us to create many objects that behave essentially in the same manner. We can use these methods to model real world objects. I wanted to create an Investments class that would model investing ones money into some low risk and high-risk options, to showcase an imaginary portfolio of assets and further demonstrate how classes can model a real-world object. In creating my class I wanted to choose arguments that would simulate a real life-investing portfolio. I chose some low risk options: annuities, bonds; and some high-risk options: collectibles (art, valuables), real estate (fluctuating markets), stocks and futures (increased volatility). This is a generalized framework but allows us create a class so that you can see how you would choose to spread your wealth across these different options depending on your risk aversion(if you are a risk taker or do not like risk at all). When discussing attributes we are really talking about getter and setter methods. We can use attr_reader, attr_writer, and attr_accessor in the code under our class declaration. The attr_reader is a getter, which means it returns a value or state outside of the class, but doesn’t change it. An attr_writer is a setter, which means you can change the value of the variable outside of the class, but it is not legible. Next we have attr-accessor, which is both a getter and a setter.
